# Reflection – Assignment 5

This assignment focused on translating system requirements into practical system interactions through use case modeling and test case development.
One of the main challenges encountered was ensuring that all use cases were directly aligned with the functional requirements defined in Assignment 4. It required careful mapping between stakeholder needs, system capabilities, and user interactions.
Another challenge was balancing simplicity and completeness. While use cases should remain clear and understandable, they must also capture all relevant system behaviors, including alternative and exception flows.
Developing test cases introduced a different perspective, shifting focus from system design to system validation. It was necessary to think critically about how each requirement would be tested and verified in a real-world environment.
Non-functional requirements, particularly performance and security, required scenario-based testing rather than simple step validation. This highlighted the importance of considering system behavior under stress and ensuring secure access control.
Overall, this assignment reinforced the importance of traceability between stakeholders, requirements, system design, and testing. It demonstrated how structured modeling improves system clarity, reduces ambiguity, and supports successful implementation.
The process also emphasized the role of proper documentation in software engineering, ensuring that both technical and non-technical stakeholders can understand the system and its expected behavior.
